Core

Dynamic Position Sizing
Adaptive sizing based on Kelly criterion (conservative 2% of capital). Streak multipliers (loss 0.5x, win 1.2–1.5x), drawdown protection, confidence scaling, XGBoost correctness penalty, asymmetry penalty for skewed win/loss ratios.
GET /bet-sizing Kelly fraction

XGBoost Direction Model
11 input features: confidence, fear & greed index, RSI-14, technical score, cyclic hour/day encoding, session label, regime label. LLM direction must agree with XGBoost prediction (dual-gate). Fail-open if model unavailable.

ACE — Adaptive Calibration Engine
NEW
Dynamic confidence threshold that adjusts based on recent performance window. Recalculates on every ghost signal batch. Prevents over-trading during drawdown phases. Separate thresholds per direction (UP/DOWN).
GET /adaptive-estimate 38 unit tests
Ghost Signal Verification
Autonomous evaluation of skipped signals at T+30min using Binance 1-minute klines. Generates training data without risking capital. Calculates ghost_correct for each SKIP/ALERT signal, enabling model improvement from paper trades.
POST /ghost-evaluate T+30min exit
Auto-Retrain Pipeline
Nightly at 03:00 UTC: build dataset (including ghost signals) → train XGBoost + correctness model → hot-reload in memory → refresh calibration tables + dead hours + ACE thresholds. On-demand retrain available (rate-limited 1/6h).
POST /auto-retrain POST /force-retrain POST /reload-calibration
Market Regime Detection
Real-time regime classification using 4-hour Binance OHLCV data. ATR%, trend strength → labels: RANGING (0), TRENDING (1), VOLATILE (2). Used as XGBoost feature and for dead hours recalibration per regime.
GET /btc-regime

Risk Management
6 GUARDS
Circuit Breaker
5 consecutive losses triggers automatic bot pause. Manual or automatic resume with _RESUMED_AT tracking. Prevents cascade losses during adverse market conditions.
Dead Hours Filter
Skips trading during UTC hours with historically low win rate (<45%). Recalculated automatically by the nightly retrain pipeline based on latest performance data.
Macro Event Guard
Blocks trading 2 hours before USD high-impact economic events (NFP, FOMC, CPI). Live macro calendar polling. Prevents entering positions during unpredictable volatility spikes.
GET /macro-guard
Anti-Stacking
Maximum 2 concurrent open positions. Pyramid only allowed if existing position P&L > 0% or position age < 10 minutes. Pyramid size = 0.5× base size.
Price Drift Detection
Rejects trade if current market price has drifted significantly from the signal price. Prevents stale signal execution in fast-moving markets.
Rate Limiting
Thread-safe sliding window (no Redis dependency). Global: 100/min, bet endpoint: 50/min, Telegram: 10/min. Per-IP rate limiting on authentication endpoints.

On-Chain Audit Trail
POLYGON PoS
Immutable Prediction Commitment
Every signal is registered on Polygon PoS before trade execution. Smart contract BTCBotAudit.sol stores keccak256 hash, direction, and confidence via commitPrediction(). After close, outcome is resolved via resolvePrediction(). Anyone can verify on Polygonscan that commitment timestamp precedes execution.
POST /commit-prediction POST /resolve-prediction GET /on-chain-audit Polygonscan verified
News Fact-Check Registration
SHA-256 hash of news headlines registered on Polygon before publication. Provides immutable timestamp proof that a news item existed before any trading decision. Prevents retroactive claim of information advantage.
POST /news-fact-check
Verifiability Guarantees
Any observer can independently verify: (1) commitPrediction was called BEFORE Kraken fill, (2) resolvePrediction matches Supabase outcome, (3) wallet signatures are authentic. Contract is append-only — no modifications possible after commitment.
